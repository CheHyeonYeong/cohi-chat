# cohi-chat에서 내가 한 일 정리

## 프로젝트 한 줄 요약

Spring Boot + React 기반의 1:1 미팅 예약 서비스에서 백엔드 핵심 기능 개발, 외부 서비스 연동, 배포 인프라 전환, 프론트엔드 UX 개선, CI/CD 보강을 맡아 진행했다.

## 내가 주로 담당한 영역

- 예약, 회원, 파일 관리 등 백엔드 기능 구현
- Google Calendar, AWS S3, AWS Secrets Manager 연동
- 데이터베이스 및 배포 구조 전환
- 홈/로그인/회원가입/호스트 등록 등 프론트엔드 UX 개선
- 테스트 코드, 운영 문서, CI/CD 워크플로우 보강

## 주요 기여 정리

### 1. 예약/회원 도메인 핵심 기능 구현

- 회원 탈퇴 시 미래 예약을 자동 취소하고, 호스트 예약의 경우 Google Calendar 이벤트까지 정리하는 기능을 구현했다.
- 탈퇴 전 영향 범위를 확인할 수 있는 API를 추가해, 사용자가 예약 취소 영향을 이해한 뒤 탈퇴할 수 있도록 설계했다.
- 예약 취소 사유를 저장하도록 모델을 확장해, 운영 관점에서 취소 원인 추적이 가능하게 만들었다.

근거:

### 2. Google Calendar 연동 범위 확장

- 백엔드 DTO 검증, 프론트 입력 검증, 안내 문구와 플레이스홀더까지 함께 수정해 실제 사용 흐름이 끊기지 않도록 반영했다.
- 관련 백엔드/프론트 테스트를 추가해 입력 케이스를 보강했다.

## 현재 진행 중인 작업

아직 커밋되지 않았지만, 현재 작업 트리 기준으로는 운영 관측성(Observability) 강화를 진행 중이다.

- `Request ID`를 생성해 로그와 응답 헤더에 함께 남기도록 필터를 추가
- `@Observed`와 Micrometer를 적용해 회원가입, 로그인, 탈퇴, 예약 생성/취소 등의 주요 흐름을 계측
- HTTP 요청/응답 로그, 느린 요청 경고, 느린 SQL 로깅을 위한 설정 추가
- Actuator 노출 범위를 확장하고 health 상태를 주기적으로 확인하는 로깅 구성 추가
- Fluent Bit, Vector, PostHog 연계를 위한 로그 수집 설정 초안 작성

## 포트폴리오/면접용 짧은 버전

cohi-chat 프로젝트에서는 Spring Boot와 React 기반 1:1 미팅 예약 서비스를 개발하면서, 예약/회원 도메인 기능과 Google Calendar, S3, AWS Secrets Manager 같은 외부 서비스 연동을 주도했다. 특히 회원 탈퇴 시 예약 자동 취소 및 캘린더 정리, Google Calendar 개인 캘린더 연동, S3 Pre-signed URL 기반 파일 처리, SQLite에서 PostgreSQL로의 마이그레이션, S3 + CloudFront 기반 프론트 배포 전환을 담당했다. 프론트엔드에서는 홈/로그인/회원가입 UX를 개선하고, 검증 훅과 상태 피드백을 도입해 사용자 흐름을 다듬었으며, 테스트와 문서를 함께 정리해 운영 안정성과 협업 생산성까지 챙겼다.

## 면접에서 강조하기 좋은 포인트

- 기능 구현만 한 것이 아니라, 실제 운영 환경에서 필요한 배포 구조와 보안 설정까지 다뤘다.
- 백엔드, 프론트엔드, 인프라를 분리하지 않고 서비스 흐름 전체를 연결해서 개선했다.
- 단순 CRUD보다 "탈퇴 시 예약 정리", "개인 캘린더 연동", "직접 다운로드 구조 전환"처럼 정책과 설계 판단이 필요한 작업을 맡았다.
- 기능 추가 후 테스트, 문서, 워크플로우까지 정리하는 방식으로 작업했다.
